# 实时监控界面参数说明

本文说明“实时监控”页面中视频画面的检测标注、右侧检测结果以及检测参数的实际含义。内容以当前代码实现为准。

## 1. 车辆检测框上的信息

一辆车被识别后，画面上的标注大致如下：

```text
┌─ #17 小汽车 86%
│  京A12345 (92%) ✓ 白名单
│
│        车辆区域
└────────────────
```

### 1.1 `#17`：车辆跟踪 ID

- 来自 ByteTrack 的本地跟踪编号。
- 用于在连续视频帧中判断“这是不是同一辆车”。
- 编号只在当前摄像头的当前跟踪周期内有意义。
- 切换视频源、重置跟踪或跟踪目标丢失后，编号可能重新分配。
- 它不是车牌号，也不是跨摄像头永久唯一的车辆编号。
- 当跟踪器没有返回有效 ID 时，标签中不会显示 `#编号`。

### 1.2 `小汽车`：车辆类别

当前只保留 COCO 数据集中的四类道路车辆：

| 中文名称 | 模型类别 | COCO 类别 ID |
| --- | --- | ---: |
| 小汽车 | `car` | 2 |
| 摩托车 | `motorcycle` | 3 |
| 公交车 | `bus` | 5 |
| 卡车 | `truck` | 7 |

其他类别不会进入车辆结果列表。

### 1.3 `86%`：车辆 YOLO 检测置信度

- 表示 YOLO 模型认为该区域属于相应车辆类别的可信程度。
- 取值范围是 `0%` 到 `100%`。
- 它受右侧“车辆阈值”控制。
- 它不是车牌 OCR 置信度。

例如“车辆阈值”为 `0.50` 时，低于约 `50%` 的车辆候选框会在模型预测阶段被过滤。

### 1.4 车辆框颜色

| 颜色 | 含义 |
| --- | --- |
| 绿色 | 已识别到车牌，并且车牌匹配白名单 |
| 橙色 | 已识别到车牌，但没有匹配白名单 |
| 蓝色 | 已检测到车辆，但当前没有识别到可用车牌 |

颜色反映的是最新一次检测结果。检测间隔大于 1 帧时，中间帧会继续绘制缓存结果。

## 2. 车牌信息条

识别到车牌后，车辆框附近会显示类似：

```text
京A12345 (92%) ✓ 白名单
```

### 2.1 `京A12345`：OCR 车牌文本

处理流程为：

1. 车牌检测模型在整帧中定位车牌和四个角点。
2. 根据角点对车牌图像进行透视矫正。
3. CRNN 模型识别车牌字符。
4. 根据车牌中心点，将车牌关联到包含它的车辆框。
5. 同一车辆存在多个候选车牌时，选择 OCR 置信度最高的一条。

### 2.2 `(92%)`：车牌 OCR 置信度

- 这是字符识别结果的置信度，不是车牌框检测置信度。
- 代码对每个有效字符位置取最大 Softmax 概率，再计算这些字符概率的平均值。
- 它受右侧“车牌阈值”控制。
- 低于“车牌阈值”的 OCR 文本不会进入最终车牌结果。

当前车牌框检测模型自身使用固定的检测阈值 `0.30`，界面暂时没有提供单独的“车牌框检测阈值”设置。

### 2.3 `白名单` / `非白名单`

- `✓ 白名单`：识别车牌通过白名单匹配。
- `✗ 非白名单`：识别到车牌，但没有通过白名单匹配。

白名单依次支持：

1. 完整车牌精确匹配。
2. 白名单车牌前缀匹配。
3. 去除空格、连字符、点号后的归一化匹配。

## 3. 视频画面统计

画面左上角的信息面板显示最新检测周期的统计，例如：

```text
车辆检测: 4 辆 | 车牌识别: 2 个
白名单匹配: 1/4 | 白名单总数: 20
```

| 字段 | 含义 |
| --- | --- |
| 车辆检测 | 当前最新检测帧中保留的车辆数量 |
| 车牌识别 | 当前结果中通过 OCR 阈值并成功关联到车辆的车牌数量 |
| 白名单匹配 | 当前车辆中匹配白名单的数量 / 当前车辆总数 |
| 白名单总数 | 白名单配置中保存的记录总数 |

这些数值是当前检测结果，不是应用启动以来的累计值。

检测间隔大于 1 时，未执行新推理的中间帧会继续绘制上一次检测框，统计面板可能带有“缓存”标记。缓存帧不会重新计算车辆位置和车牌。

## 4. 视频区域下方指标

| 指标 | 含义 |
| --- | --- |
| 车辆 | 最新检测结果中的车辆数 |
| 车牌 | 最新检测结果中有有效车牌文本的车辆数 |
| 白名单 | 最新检测结果中匹配白名单的车辆数 |
| 检测状态 | 未启用、等待加载、正在加载、模型已就绪或检测异常等状态 |

视频标题栏还会显示当前图像分辨率和发布帧率。发布帧率是浏览器收到的标注视频帧速率，不等同于模型每秒推理次数。

## 5. 右侧“检测结果”页签

| 列 | 实际含义 |
| --- | --- |
| 目标 | 跟踪 ID 和车辆类别，例如 `#17 小汽车` |
| 车牌 | OCR 识别出的车牌；没有有效结果时显示 `--` |
| 置信度 | 当前实现显示车辆 YOLO 置信度，不是车牌 OCR 置信度 |
| 状态 | `白名单`、`未匹配`或`无车牌` |

“当前画面”右侧的数字是最新检测结果中的车辆条目数。

> 注意：画面车牌信息条会显示车牌 OCR 置信度，但右侧结果表的“置信度”列目前显示车辆置信度。两者不能直接比较。

## 6. 右侧“检测参数”页签

### 6.1 推理设备

指定车辆检测模型和车牌识别模型运行在哪个计算设备上。

| 选项 | 含义 | 特点 |
| --- | --- | --- |
| CPU | 使用处理器推理 | 兼容性最好，通常速度最慢 |
| GPU: NVIDIA 型号 | 使用对应的 CUDA GPU，例如 `cuda:0` | 通常速度最快，需要可用的 CUDA/PyTorch 环境 |
| GPU (Apple MPS) | 使用 Apple Silicon GPU | 仅在支持 MPS 的设备上出现 |

界面只列出后端实际检测到的可用设备。切换设备后，后台会重新创建并加载检测处理器；模型加载期间检测状态会显示加载信息。

### 6.2 车辆阈值

| 属性 | 值 |
| --- | --- |
| 参数名 | `yolo_threshold` |
| 可选范围 | `0.05` 到 `1.00` |
| 调整步长 | `0.05` |
| 默认值 | `0.50` |
| 作用对象 | YOLO 车辆检测结果 |

- 调高：保留的车辆框更可信，误检通常减少，但可能漏掉远处、遮挡或模糊车辆。
- 调低：更容易检测弱目标，漏检通常减少，但误检可能增加。
- 该阈值会直接传入 YOLO 的 `conf` 参数。

### 6.3 车牌阈值

| 属性 | 值 |
| --- | --- |
| 参数名 | `lpr_threshold` |
| 可选范围 | `0.05` 到 `1.00` |
| 调整步长 | `0.05` |
| 默认值 | `0.70` |
| 作用对象 | CRNN 输出的车牌 OCR 文本 |

- 调高：过滤更多低可信车牌文本，错字和误识别通常减少，但车牌识别数量也会下降。
- 调低：输出更多车牌文本，召回率提高，但错误车牌可能增加。
- 该参数不改变车牌框检测模型的 `0.30` 检测阈值。

### 6.4 检测间隔

| 属性 | 值 |
| --- | --- |
| 参数名 | `interval` |
| 可选范围 | `1` 到 `60` 帧 |
| 默认值 | `5` 帧 |
| 作用 | 每隔多少个视频帧执行一次完整车辆和车牌推理 |

示例：输入视频为 30 FPS 时：

| 检测间隔 | 理论检测频率 | 说明 |
| ---: | ---: | --- |
| 1 帧 | 每秒最多 30 次 | 响应最快，GPU/CPU 负载最高 |
| 3 帧 | 每秒最多 10 次 | 较实时，负载适中 |
| 5 帧 | 每秒最多 6 次 | 当前默认值 |
| 10 帧 | 每秒最多 3 次 | 负载较低，框位置更新更慢 |

实际频率还受到模型推理耗时、视频解码速度和硬件性能影响。两个推理周期之间会使用上一次结果绘制缓存框。

### 6.5 应用参数

- 调整推理设备、车辆阈值、车牌阈值或检测间隔后，必须点击“应用参数”。
- 点击后，前端通过 `/api/detection/settings` 将整组参数发送给后端。
- 参数在后续检测周期生效；已经缓存的结果可能短暂保留到下一次完整推理。
- 调整滑块但不点击“应用参数”只会改变页面上的数值，不会更新后端。

## 7. 智能检测开关

“智能检测”开关独立于“应用参数”按钮，并且会立即提交：

- 开启：后台按当前设备、阈值和检测间隔加载并运行模型。
- 关闭：停止执行新的检测推理，并清空后端当前检测结果。
- 视频解码和画面播放不受该开关影响。

模型第一次开启或切换设备时需要加载权重，检测状态会依次显示“等待加载模型”“正在加载模型”和“模型已就绪”。

## 8. 推荐起始配置

以下数值是调试起点，不是固定标准，最终应按摄像头距离、光照、车速和硬件性能实测。

| 场景 | 车辆阈值 | 车牌阈值 | 检测间隔 |
| --- | ---: | ---: | ---: |
| 常规监控 | 0.50 | 0.70 | 5 |
| 优先减少误检 | 0.65 | 0.80 | 3 至 5 |
| 远距离或弱光，优先减少漏检 | 0.30 | 0.55 | 2 至 3 |
| 性能不足，优先保证视频流畅 | 0.50 | 0.70 | 8 至 15 |

建议一次只调整一个参数，并分别观察车辆框数量、有效车牌数量和错误车牌比例。

## 9. 常见误解

1. **车辆框百分比不是车牌置信度。** 它是车辆 YOLO 置信度。
2. **车牌阈值不是车牌框检测阈值。** 它过滤的是 OCR 字符结果。
3. **跟踪 ID 不是永久车辆 ID。** 它可能在切流或丢失跟踪后变化。
4. **发布 FPS 不是推理 FPS。** 完整推理只在满足检测间隔时执行。
5. **滑块不是即时保存。** 除智能检测开关外，参数需要点击“应用参数”。
6. **阈值变化不会重算旧帧。** 新阈值从后续检测周期开始影响结果。

## 10. 代码对应关系

| 功能 | 实现位置 |
| --- | --- |
| 车辆类别、YOLO 阈值和 ByteTrack ID | `vehicle_detector.py` |
| 车辆框标签、车牌条和颜色 | `detection_processor.py`、`draw_utils.py` |
| OCR 置信度计算与过滤 | `lpr_recognizer.py` |
| 检测间隔、阈值动态应用和缓存结果 | `backend/video_stream.py` |
| 右侧参数表单与结果表 | `frontend/index.html`、`frontend/js/app.js` |
